[4] 번역 : 컴파일러(Compiler) vs 인터프리터(Interpreter) 언어
[4-1] 컴파일러(한꺼번에 기계어로 번역) ex. 떡볶이-밀키트 -> 정해져있음(수정x)

[4-2] 인터프리터(코드를 한줄씩 읽어가며 즉시 실행) ex. 떡볶이-장보기(요리) -> 시행착오 가능함
         장점 : 실시간수정,쉬운 디버깅 | 단점 : 컴파일러에 비해 실행속도가느림
[4-3] 하이브리드 : 컴파일러,인터프리터 장점 융합

[6] c언어 vs 파이썬 차이 
    ex. | c언어 int a=10;  /  파이썬 a=10 |
    =>[c언어] int b=10; 지정시 int의 공간(4byte)을 할당하고 변수(10)를 저장, a랑 독립적인 메모리 공간을 가짐
    =>[파이썬] b=10 지정시 이미 10이라는 메모리 주소를 가진것을 a도알고 b에게도 알려줘서 같은 메모리 주소를 공유하게됨(내가 임의의 공간을 지정할수없다는 단점이있음)

[7] 프로그래밍/코딩의 시작
-> 어떤 변수를 지정할것인가? 변수에 어떤값을 지정할것인가? 관계를 어떻게 정의할것인가?
** 변수 : 임의의 데이터 값을 일시적으로 저장하기 위해 사용하는 메모리의 한 영역 
    -> 코드를 동작시키는데 필요한 데이터를 저장하기 위해 내가 잠깐 메모리상의 공간을 쓴다
   ex1) 사과가 10개가있다. 사과 1개의 가격은 1000원이다. 총비용은 얼마인가?
    -> 변수(사과)의 가격을 지정. 사과의 총개수를 지정. 총비용의 관계를 지정(사과*총개수)
   ex2) 장바구니안에 과일a,b,c가있다. 과일a,b,c의 가격을 각각 a1,b1,c1이라 하자. 총비용은 얼마인가?
    -> 변수(과일a,b,c)지정. 과일 a,b,c의 가격(a1,b1,c1)지정. 총비용=0에서 장바구니안의 물건을 하나씩 꺼낼때마다 더하기(** 랜덤으로 하나씩 꺼냈는데 a1,b1,c1순으로 차례대로 뽑았다고 가정. 이 순서는 랜덤임)(총비용=0 -> 0=a1 -> a1=a1+a1 -> a1+a1=a1+a1+a1 -> a1+a1+a1=a1+a1+a1+b1 -> a1+a1+a1+b1=a1+a1+a1+b1+c1)
*** ex1), ex2)차이 : ex1) c=a*b / ex2) c=Σ
**** 코드 작성시 *,+중 +를 많이써라(*,+ 처리속도가 차이가 많이난다)(추후 염두에두기)
